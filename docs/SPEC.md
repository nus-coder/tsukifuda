# ツキフダ 〜月夜の十二番勝負〜 仕様書

オリジナル2人対戦カードゲームのWebアプリ。CPU対戦（初心者／ハード）とオンライン1対1対戦（PeerJS）に対応する。
1ゲーム10〜20分。ルールは3分で覚えられるが、カードカウンティングと読み合いで奥が深いことを目指す。

## 1. ゲームコンセプト

- 両プレイヤーは **まったく同じ12枚の妖怪カード**（パワー0〜11、各1枚、固有能力つき）を手札に持つ。
- 場には12枚の **月齢カード**（そのラウンドの得点＝月光点）が **最初から全ラウンド分公開** されている。
- 毎ラウンド、両者が同時に手札から1枚を伏せて出し、同時公開。勝った方がその月齢の月光点を得る。
- 使ったカードは戻らない。**相手の残り手札は常に全部わかる**（同一デッキだから）。
  → 隠された情報は「今この瞬間、相手が何を出すか」だけ。純粋な読み合い（ヨミ）のゲーム。
- 引き分けのラウンドは月光点が**持ち越し（ポット）**になり、次のラウンドの価値が膨らむ。
- 12ラウンド終了時に月光点が多い方の勝ち。

### 面白さの核

1. **完全公開情報の読み合い**：「相手には11が残っている。この満月に切ってくるか？」という会話が自然に生まれる。
2. **月齢トラックの計画性**：高得点ラウンドがいつ来るか最初からわかるので、リソース配分の長期戦略がある。
3. **逆転ルール**：新月（弱い方が勝つ）と月蝕（能力無効）が読み合いの軸を反転させる。
4. **ポットの膨張**：妖狐や同カード対決で引き分けを量産すると、1ラウンドに8点以上かかる大勝負が生まれる。

## 2. カードリスト（各プレイヤー12枚、同一構成）

| パワー | 名前 | 能力 |
|---|---|---|
| 0 | ねずみ小僧 | 相手のパワーが**10以上**なら勝つ（下剋上）。 |
| 1 | 提灯おばけ | 負けたとき、このラウンドの月光点は相手に渡らず**持ち越し**になる（灯を消す）。 |
| 2 | 猫又 | **相手カードの能力を無効化**する（パワーだけの勝負にする）。 |
| 3 | 河童 | 負けたとき、自分は**1点**得る（皿の水）。 |
| 4 | 天狗 | 勝ったとき、**追加で1点**得る。 |
| 5 | 妖狐 | このラウンドは**強制的に引き分け**になる（月光点は持ち越し）。 |
| 6 | 侍の霊 | 勝ったとき、**次のラウンド**自分の出すカードのパワー**+2**。 |
| 7 | 巫女 | パワーが**同点なら勝つ**。 |
| 8 | 人狼 | **満月**のラウンドならパワー**+5**（=13）。 |
| 9 | 大蛇 | 勝ったとき、相手の得点を**1点奪う**（相手が0点なら奪えない）。 |
| 10 | 竜神 | 能力なし。純粋な力。 |
| 11 | 月読 | 能力なし。最高のパワー。ただし、ねずみ小僧と新月には要注意。 |

## 3. 月齢カード（12枚、毎ゲームシャッフルして順番を決める）

| 月齢 | 枚数 | 月光点 | 効果 |
|---|---|---|---|
| 三日月 | 4 | 1 | なし |
| 半月 | 4 | 2 | なし |
| 満月 | 2 | 3 | 人狼のパワー+5が発動する |
| 新月 | 1 | 3 | **パワーの低い方が勝つ** |
| 月蝕 | 1 | 4 | **全カードの能力が無効**（パワーだけの勝負） |

合計25点。順番はゲーム開始時にシャッフルされ、**全12ラウンド分が最初から公開**される。

## 4. ラウンド解決手順（エンジンの正）

1. **月蝕チェック**：月蝕ラウンドなら両カードの能力をすべて無効化。
2. **猫又**：（無効化されていなければ）相手カードの能力を無効化。両者猫又は同カード対決なので自動的に引き分け。
3. **パワー修正**：人狼（満月なら+5）、侍の霊の持ち越しバフ（前ラウンドで侍が勝っていれば+2）を適用。
4. **妖狐**：能力が生きていれば強制引き分け（→手順7へ。巫女でも覆せない。パワー同点扱いではないため）。
5. **勝敗判定**：
   - ねずみ小僧の能力が生きていて相手の修正後パワーが10以上 → ねずみ小僧の勝ち。
   - 新月ラウンド → 修正後パワーの**低い方**が勝ち。それ以外 → 高い方が勝ち。
   - 同点の場合：片方だけ巫女の能力が生きていれば巫女の勝ち。それ以外は引き分け。
   - **同カード対決（ミラー）は必ず引き分け**（パワー同点・能力が相殺されるため自然にそうなる）。
6. **得点**：勝者は「月光点＋ポット」を獲得。引き分けならポットに月光点を積む。
7. **得点系能力**（無効化されていなければ、この順で）：
   - 提灯おばけ（負け）：勝者が得るはずだった月光点＋ポットをポットへ戻す（勝者は0点。天狗ボーナス等は別途入る）。
   - 天狗（勝ち）：+1点（銀行から）。
   - 河童（負け）：+1点（銀行から）。
   - 大蛇（勝ち）：相手から1点奪う（相手の得点が0なら不発）。
8. **侍の霊**：このラウンドで侍の霊が勝ったら、その持ち主の次ラウンドのカードにパワー+2を予約。
9. 最終ラウンドで持ち越しが発生した場合、そのポットは消滅する（月に還る）。

### 勝敗
12ラウンド終了時に得点が多い方の勝ち。同点なら引き分け。
（天狗・河童のボーナスと大蛇の強奪があるため、合計は25点からずれうる）

## 5. CPU（AI）

- 実装は `js/ai.js`。エンジン（`js/engine.js`）は純粋関数なのでAIがそのままシミュレーションに使う。
- **初心者（novice）**：月光点に見合ったパワーのカードを出す「価値マッチング」＋大きなランダムノイズ。たまに明らかな悪手も打つ。能力はほぼ考慮しない。
- **ハード（hard）**：
  1. 自分の残り手札 × 相手の残り手札の**利得行列**を作る。各セルは「そのラウンドの得点差＋残カードの将来価値差」（将来価値は残りラウンドの月光点分布でスケール）。
  2. **後悔マッチング（regret matching）を数百回**回して混合戦略（ナッシュ均衡近似）を計算。
  3. 得られた確率分布からサンプリングして出すカードを決める。→ 読まれにくく、かつ搾取されにくい。

## 6. オンライン対戦

- **PeerJS**（CDN、公式クラウドブローカー使用、サーバー実装不要）でWebRTCのP2P接続。
- ホストが「部屋を作る」→ 5文字の合言葉コードが発行される。ゲストはコードを入力して参加。
- ホストが月齢デッキをシャッフルして初期状態を送信。以後は両クライアントが決定論的なエンジンで同一進行。
- **チート防止（コミット＆リビール）**：各ラウンド、まず自分の選択の SHA-256 ハッシュ（選択+乱数ノンス）を送り合い、両者のコミットが揃ってから平文を公開。相手の選択を見てから自分の手を変えることはできない。
- 切断時はメッセージを出してタイトルへ戻る。

## 7. 技術構成

- ビルド不要の静的Webアプリ。`index.html` をブラウザで開くだけで動く（オンライン対戦もCDN読み込みのみ）。
- ファイル構成:
  - `index.html` — 全画面（タイトル／ルール／ゲーム／オンラインロビー／結果）
  - `css/style.css` — 全スタイル（夜空・和風テーマ）
  - `js/cards.js` — カード定義＋インラインSVGイラスト（12体）＋月齢定義
  - `js/engine.js` — 純粋なゲームロジック（状態生成・ラウンド解決）。UIにもAIにもオンラインにも依存しない
  - `js/ai.js` — 初心者／ハードAI
  - `js/ui.js` — DOM描画・アニメーション
  - `js/online.js` — PeerJS接続・コミット＆リビール
  - `js/main.js` — 画面遷移とゲームループの束ね
- カードイラストはすべてインラインSVG（外部画像・外部フォントなし）。

## 8. 画面仕様

- **タイトル**：ロゴ、モード選択（CPU初心者／CPUハード／オンライン対戦／遊び方）。
- **ゲーム画面**：
  - 上段：相手の残り手札（表向き・小さく）、相手得点。
  - 中段：月齢トラック（12個、現在をハイライト、済みはグレー）、ポット表示、カード公開エリア（同時めくり演出）。
  - 下段：自分の手札（イラスト付き、ホバーで能力表示）、自分得点、決定ボタン。
  - ログ（直近の解決内容をテキストで）。
- **結果画面**：勝敗、最終得点、再戦／タイトルへ。

## 9. 演出・付加機能

- **効果音**（`js/sound.js`）：Web Audio API で全SFXを合成（音声ファイル不要）。カードめくり・勝利/敗北・引き分け・妖狐の妖術・提灯の吹き消し・大蛇の強奪・満月の遠吠え・月蝕ドローン・コインなど。右上の🔊ボタンでミュート切替（localStorage永続）。
- **固有演出**：得点のフロートテキスト、賭け金5点以上の決着で金色フラッシュ＋画面シェイク、月蝕/満月/新月のラウンド開始フラッシュ。
- **絵文字リアクション**（オンライン戦のみ）：😼🔥😱🙏😆🌕 の6種を対戦中に送り合える。連打防止で1.5秒に1回。プロトコルは `{t:'emote', e:index}`。
- **戦績記録**：`localStorage`（キー `tsukifuda-stats`）にモード別（novice/hard/online）の勝敗分を記録し、タイトル画面に表示。ストーリーモードは戦績に含めない。
- **タイトル演出**：月の浮遊アニメ、流れ星、蛍のパーティクル。
- **BGM**（`js/sound.js`）：Web Audio による**生成音楽**。陰旋法（A・B♭・D・E・F）の琴風プラックをランダムに紡ぎ、低音ドローンとエコーを重ねる。音源ファイル不要・ループなし（同じフレーズは二度と鳴らない）。右上🎵ボタンと中断メニューでON/OFF（localStorage `tsukifuda-bgm`）。ブラウザの自動再生制限のため初回クリック後に開始。
- **パーティクルエンジン**（`js/fx.js`）：依存ゼロの Canvas 2D。勝利の金色バースト、大勝負（5点以上）の紙吹雪シャワー、妖狐の霧粒子、人狼の咆哮リング、大蛇の強奪（点が弧を描いて飛ぶ）、タイトルの蛍。`FX.burst/shower/mist/ring/steal/setAmbient` を UI から座標付きで呼ぶ。
- **キャッシュバスト**：`index.html` の script/css には `?v=N` を付与。ファイルを更新したら N を上げること。
- **カード固有のビジュアル演出**（`ui.js` の revealRound 内）：
  - 妖狐：強制引き分け時、勝負エリアに青い霧が流れる。
  - 人狼：満月で発動時、カードが赤く発光して震える変身アニメ＋「+5」表示。
  - ねずみ小僧：パワー10以上を倒すと跳び上がる「下剋上！」演出。
  - 猫又：無効化された相手カード（能力持ちのみ）に「封」の朱印スタンプ。
- **遊び方**：4ページ構成のページ式チュートリアル（1.基本 → 2.月齢 → 3.ポットと処理順 → 4.カード一覧）。ミニカードと月齢アイコンの図解つき。タイトルからも対戦中の中断メニューからも開ける。
- **中断メニュー**：対戦中の☰ボタンで開く。「対戦に戻る／ルールを見る／効果音切替／投了する（2段階確認）」。投了はCPU・オンライン戦では負けとして戦績に記録され、オンライン戦では相手に切断が通知される。

## 10. ストーリーモード「月夜の五番勝負」

- 実装は `js/story.js`（ボス定義）＋ `main.js`（進行）。進行度は `localStorage`（キー `tsukifuda-story`、クリアしたボス数）。
- 5人のボスを順に倒す。未クリアのボスはロック表示（名前は「？？？」）。全クリアでバナー表示。
- 対戦前にボスの会話デモ（クリック送り）。勝利/敗北後もボスの台詞が結果画面に出る。
- 各ボス（名前 / AI / ギミック）:
  1. 見習いのチュウタ（novice / 標準ルール）
  2. 沼の大親分ガタロ（mid / 開始時ポット3点）
  3. 猫又の女将ミケ（mid / 月蝕×2の月齢構成）
  4. 月牙のガロウ（hard / 満月×4の月齢構成）
  5. 月読命（hard / 新月×2＋月蝕×2の月齢構成）
- AI「mid」は35%の確率で novice の手を混ぜた hard。カスタム月齢構成は12枚・合計奇数（引き分け防止）で設計する。
- エンジン側の対応: `ENGINE.shufflePhases(pool)`（カスタム構成）と `ENGINE.newGame(phases, startPot)`（初期ポット）。
