# ツキフダ — 追加アイデア集

「もしここから拡張するなら」の候補。おおよそ実装コストが低い順・効果が高い順に並べている。

## A. すぐできる磨き込み（小）

1. ~~**効果音**~~ ✅ 実装済み（Web Audio API合成、`js/sound.js`）。**BGM**（アンビエントな和風ドローン）は未実装。
2. **演出強化**：✅ 一部実装済み（得点フロート・画面シェイク・月齢フラッシュ・遠吠え等）。妖狐の霧・人狼の変身アニメなどカード固有のビジュアル演出はまだ余地あり。
3. **ターンタイマー**：オンライン戦で30秒制限＋時間切れはランダム提出。だらけ防止。
4. ~~**絵文字リアクション**~~ ✅ 実装済み（6種・レート制限つき）。テキスト定型チャットは未実装。
5. **モバイル最適化**：タップでツールチップ表示、縦持ちレイアウト。
6. **カード裏拡大表示**：長押し/右クリックでカードをズームしてイラストとフレーバーを堪能できるモード。
7. **色覚対応**：勝敗の金/グレーに加えて記号（○●）を常時併記。

## B. ゲームプレイの深化（中）

1. **ランク戦とレーティング**：✅ モード別勝敗のローカル記録は実装済み（タイトル画面に表示）。連勝数・レーティング化は未実装。
2. **リプレイと棋譜**：12ラウンドの選択を記録して見返せる。「あそこで月読を切っていれば…」の検討会ができる。URLに棋譜を埋め込んで共有。
3. **CPUの最強化（月読AI）**：終盤（残り5枚以下）は完全な後ろ向き帰納法で厳密な均衡を計算。序中盤も抽象化した先読みを入れる。
4. **CPUの人格**：「攻撃的な鬼AI」「妖狐を多用するトリックスターAI」など、混合戦略の重みに個性を持たせた選択式CPU。
5. **ハンデ戦**：格上相手に先取り点ハンデ、または片方だけ月齢を1枚多く見られる等。
6. **デイリー月齢**：日替わりシード（日付から生成）で全プレイヤー同じ月齢配列に挑戦。CPU戦のスコアを競う。

## C. カードプールの拡張（中〜大）

現在は固定12枚。ここを拡張するとTCG的な奥行きが出る。

1. **ドラフト構築**：全18〜24種から交互ピックで12枚を構築してから対戦。同一デッキ前提が崩れるので「相手の手札が丸見え」の性質が変わる点に注意（ドラフトは公開で行えば性質を維持できる）。
2. **追加カード案**：
   - 「鏡女（パワー6）」：相手が出したカードのコピーになる（＝常にミラー引き分け）。
   - 「砂かけ婆（パワー3）」：このラウンドの月齢効果を反転（新月なら高い方勝ち）。
   - 「一反木綿（パワー4）」：負けたら次のラウンドの月光点を+1。
   - 「ぬらりひょん（パワー9）」：勝っても月光点を取らず、代わりに相手の任意のカード1枚を公開で封印（次ラウンド使用不可）。
   - 「雪女（パワー7）」：勝ったら相手の次ラウンドのパワー-2（凍傷）。
3. **月齢の拡張**：「朧月（両者の出したカードのパワーを入れ替える）」「流星群（このラウンドだけ能力が2回発動）」。

## D. オンライン体験の本格化（大）

1. **シグナリング自前化**：PeerJS公式ブローカー依存をやめ、Cloudflare Workers + Durable Objects や Vercel + Upstash Redis で部屋管理。切断再接続（ゲーム状態は決定論なので棋譜再送で復元可能）。
2. **観戦モード**：部屋に3人目以降は観戦者として入り、両者の手札が見える神視点で楽しむ。
3. **非同期対戦（1日1手）**：Push通知で「相手が手を出しました」。1ラウンドずつじっくり読み合う遊び方。
4. **マッチメイキング**：合言葉なしのランダムマッチ＋簡易レート。

## E. 見た目・世界観（中）

1. **イラストの高品位化**:現在は手製SVG。画像生成やイラストレーター発注で差し替え（`CARD_ART` を差し替えるだけの構造になっている）。
2. **カードのホイル加工**：CSSのconic-gradientとマウス追従によるキラカード表現。勝利数に応じてキラ化するコレクション要素。
3. **ストーリーモード**：妖怪たちを順に倒していく一人用モード。ボスごとに月齢配列に癖がある（鬼は満月だらけ、など）。
4. **多言語対応**：カード名・UIの英語化。ルールが軽いので海外フレンドとも遊びやすい。

## F. 技術的な改善

1. **PWA化**：Service Workerでオフライン対応（CPU戦はオフラインで動くべき）。ホーム画面追加。
2. **状態管理のスナップショット**：`localStorage` に進行中ゲームを保存し、リロードしても再開できるように。
3. **E2Eテスト**：Playwrightで「CPU戦を1ゲーム完走する」スモークテストをCIに。
4. **エンジンのプロパティテスト**：ランダム対局10万回で「得点が負にならない」「手札整合性」などの不変条件を検証。

## 設計メモ（拡張時の注意）

- `engine.js` は純粋関数なので、カード追加は `resolveRound` の分岐と `cards.js` の定義追加で完結する。ただし**能力の解決順序**（SPEC.md §4）に新カードを位置づけること。
- オンライン対戦は「両クライアントが決定論的に同一進行」する前提。乱数を使う能力を足す場合は、ホストが乱数値を配るか、コミット済みシードから導出すること。
- AIは `payoff()` にゲーム知識が集約されている。カードを足したら `BASE_VALUE` の調整を忘れずに。
